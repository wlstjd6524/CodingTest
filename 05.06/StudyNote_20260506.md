# 🧠 2. 핵심 로직 복기 & 새로 습득한 파이썬 무기
오늘 코테 준비의 핵심은 '리스트 컴프리헨션', 'enumerate', 'join' 을 활용하면서 코드를 압축시켜보는 기술을 접목시켜보려고 했다.

- 문자열 자체가 리스트다 : 0 ~ 9의 숫자를 알파벳 a ~ j 로 바꿀 때, 굳이 복잡한 아스키 연산 할 필요 없이 `abcdefghij[인덱스]` 형태로 직관적 매핑이 가능하다.

- 위치(Index) 가 곧 Rank : 내 점수가 몇 등인지 알기 위해서 2중 for 문으로 남들과 크기를 비교하지 않고 `sorted(reverse=True)` 로 1등부터 줄을 세운 기준표를 만들고, `.index(My Score) + 1` 을 하면 그게 내 등수임

- 톱니바퀴 순환의 마법 : `% (나머지)`
    - 길이가 서로 다른 배열을 100번, 1000번 비교해야 할 때 배열을 복사해서 늘리지 않고 현재 문제 번호 i를 패턴의 길이로 나눈 나머지(`i % len`) 를 인덱스로 쓰면 메모리 낭비 없이 배열 안에서 무한히 순회함.

- 누적 카운트는 `sum` 이랑 제너레이터 : 
    - `if 정답 : score += 1` 이라는 케이스를 버리고 이제 `sum(1 for i, a in enumerate(answers) if 정답조건)` 패턴을 쓰면 변수 선언 없이 조건에 맞는 개수를 한 줄로 카운트 할 수 있다.

- 2차원 배열의 겹침은 비트 OR 연산 으로 
    - 0 과 1로 이루어진 배열이 겹칠 때 "둘중 하나라도 1이면 1" 이라는 조건은 OR 연산이랑 완벽하게 동일함
