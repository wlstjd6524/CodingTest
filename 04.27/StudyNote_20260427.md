# 🧠 2. 핵심 로직 복기 & 새로 습득한 파이썬 무기
오늘 코테 준비의 핵심은 '수학적 / 논리적 직관 및 패턴찾기는 괜찮지만, 파이썬 내장 자료구조와 제너레이터의 효율성을 활용하지 못함 으로 요약할 수 있겠다.

- zip() 언패킹과 삼항 연산자 :
    - 묶인 데이터는 `for num, sign in zip(...)` 형태로 변수 2개에 동시에 담아(언패킹) 직관적으로 꺼내 써야 한다.
    - `if-else` 분기문은 `[참일 때 값] if [조건] else [거짓일 때 값]` (삼항 연산자) 로 1줄 압축이 가능하다.

- 제너레이터와 메모리 최적화 :
    - `sum([x for x in arr])` 처럼 리스트를 메모리에 통째로 올리는 대신, `sum(x for x in arr)` 처럼 대괄호를 벗기면 제너레이터가 되서 메모리 낭비 없이 극도로 빠른 연산이 가능하다. (코드 줄도 줄어듦)

- map() 과 문자열 순회 :
    - 파이썬 문자열은 그 자체로 순회기능이 존재함. 숫자의 각 자릿수 합을 구할 때는 빈 배열과 for 문 대신 `sum(map(int, str(x)))` 1줄로 줄일 수 있음.

1. 중복을 확인하거나 고유한 종류의 개수를 셀 때 `for` 문 내부에서 `if x == y` 와 같은 케이스를 사용 금지. 무조건 `set()` 을 떠올릴 것.
