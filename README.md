# Coding Test

코테 준비 작업물을 저장하는 공간 입니다. <br>
BOJ 서비스가 종료됨에 따라 프로그래머스 기반으로 코테 준비를 새롭게 시작합니다.

## Phase 정의

### Phase1. 추상적 논리의 구체화 및 파이썬 머슬 키우기
- 주요토픽 : 단순 구현, 문자열 파싱, 리스트/딕셔너리 제어
- 훈련방식 : 주석으로 작성한 논리를 파이썬의 여러가지 기능에서 어떤 부분으로 매핑할지 집중 훈련
- 주의사항 : 여기서 코드가 길어지고, JAVA 나 C언어 같은 구현을 이루어낸다면 아직 파이썬에 익숙하지 않은 것 이므로 파이썬 다운 코드로 리팩토링 하는 단계를 거쳐야 함.

### Phase2. 시간 복잡도 타겟팅
- 주요토픽 : 해시, 스택/큐
- 훈련방식 : 파이썬의 collections 모듈(Counter, deque, defaultdict)을 활용하여 시간 초과(TLE)를 방지 하는 훈련.
- 주의사항 : 배열(List)의 pop(0)이나 in 연산자를 남용하여 $O(N)$ 의 시간을 낭비하는 패턴을 철저히 배제

### Phase3. 상태 공간 탐색과 최적화
- 주요토픽 : 완전탐색, DFS/BFS
- 훈련방식 : 2차원 배열(격자)에서의 탐색, 재귀 함수 설계 및 탈출 조건(Base Case) 설정
- 주의사항 : 재귀의 깊이나 방문 처리(Visited)를 놓쳐 무한 루프에 빠지는 실수를 용납하지 않기


## 규칙사항
1. 각 페이즈별 정해진 승급조건을 만족해야 페이즈 승급이 가능하며, 승급조건이 만족 됐을 때 승급전 문제로 해결해보면 좋을 문제를 선정하여 조건에 맞는 해결 규칙을 달성했다면 다음 페이즈로 넘어가 학습
2. 하루에 문제는 무슨 사정이 있던 반드시 '페이즈별 3개의 문제' 는 반드시 풀어야 함. (토요일 하루만 쉬기..)
3. '3개의 문제' 를 풀 때 무조건 주석처리로 내가 문제에 대한 고민의 흔적을 남겨야 하며, 문제당 해결시간은 '20분' (페이즈가 넘어갈 수록 문제 복잡도가 올라가기 때문에 해당 부분은 추후에 고민하여 조정)
4. 문제를 다 풀면 그 날 StudyNote 작성 후 해당 레포에 Push


목표 : 코테를 주관하는 기업들을 타겟하는 문제들이 보이는데 그 문제를 한 문제라도 접근 해볼 수 있는 역량을 키우기
