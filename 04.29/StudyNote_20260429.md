# 🧠 2. 핵심 로직 복기 & 새로 습득한 파이썬 무기
오늘 코테 준비의 핵심은 '역발상' 과 '집합(Set)' 의 맹점 파악이였다.

- `set` 은 순서를 보장하지 않는다.
    - 중복을 제거하기 위해 `set` 을 씌우는 순간, 기존에 정렬해 둔 순서가 무작위로 파괴됨.
    - 정석 패턴 : 중복 제거 후 정렬이 필요하면 무조건 `sorted(set(Data))` 순서로 작성해야 함.

- 조합의 라이브러리 `itertools.combinations:`
    - 배열에서 서로 다른 n개를 뽑는 모든 경우의 수를 구할 때 이중 for 문으로 i,j 부여해서 쓸 필요가 없다라는 걸 알게됨.
    `combinations(arr, 2) 한 줄로 조합 객체를 생성할 수 있었음.

- Generator 표현식
    - `sum(a * b for a, b in zip(A, B))` 처럼 대괄호 없이 반복문을 작성하면, 메모리에 리스트를 할당하지 않고 즉시 계산만 누적해서 Return 해가지고 압도적 메모리 효율을 낼 수 있음.
